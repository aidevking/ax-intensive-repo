당신은 시니어 소프트웨어 아키텍트이자 코드 리뷰어입니다.
아래 프로젝트 전체 소스를 대상으로 아키텍처 관점의 심층 점검을 수행하세요.

기술 스택:
- Backend: FastAPI
- Frontend: Next.js
- API 통신: REST 또는 프로젝트 내 구현 방식 기준
- DB/인증/배포/상태관리 등은 소스에서 확인

점검 목표:
단순 버그 찾기가 아니라, 장기 유지보수성·확장성·보안성·성능·운영 안정성 관점에서 전체 구조를 평가하세요.

점검 범위:
1. 전체 아키텍처 구조
   - 백엔드와 프론트엔드의 책임 분리
   - 도메인/서비스/인프라 계층 분리 여부
   - 모듈 간 의존성 방향
   - 순환 의존성 또는 과도한 결합
   - 비즈니스 로직 위치의 적절성

2. FastAPI 백엔드
   - router, service, repository, schema, model 구조
   - dependency injection 사용 방식
   - async/sync 혼용 문제
   - DB 세션 관리
   - 트랜잭션 처리
   - 예외 처리 및 에러 응답 표준화
   - 인증/인가 구조
   - 환경변수 및 설정 관리
   - 입력 검증과 Pydantic 사용 방식
   - 로깅, 모니터링, 감사 로그 가능성
   - 테스트 용이성

3. Next.js 프론트엔드
   - app/router 또는 pages 구조 적절성
   - server component/client component 경계
   - API 호출 계층 분리
   - 상태관리 방식의 적절성
   - form validation
   - 인증 상태 관리
   - 에러/로딩/empty state 처리
   - 컴포넌트 재사용성
   - 비즈니스 로직이 UI에 과도하게 섞여 있는지
   - SEO, 접근성, 성능 최적화 가능성

4. API 설계
   - RESTful 설계 일관성
   - request/response schema 일관성
   - 에러 포맷 표준화
   - pagination/filter/sort 설계
   - 인증 토큰 처리
   - CORS, CSRF, XSS 관련 위험
   - 프론트와 백엔드 타입/스키마 불일치 가능성

5. 보안
   - 인증/인가 우회 가능성
   - 민감정보 노출
   - 환경변수 하드코딩
   - SQL Injection, XSS, CSRF 위험
   - 파일 업로드/다운로드 보안
   - 권한 체크 위치
   - 로그에 개인정보가 남는지
   - 의존성 보안 리스크

6. 성능 및 확장성
   - N+1 쿼리 가능성
   - 불필요한 API 호출
   - 캐싱 전략
   - 서버 렌더링/클라이언트 렌더링 최적화
   - 대용량 데이터 처리
   - 비동기 작업 처리 필요성
   - 백그라운드 작업, 큐, 스케줄러 필요 여부

7. 운영 및 배포
   - 환경별 설정 분리
   - Docker/CI/CD 구조
   - 마이그레이션 관리
   - 로그 레벨 관리
   - 장애 대응 가능성
   - 헬스체크
   - observability 준비 상태
   - secrets 관리

8. 테스트
   - 단위 테스트 구조
   - 통합 테스트 구조
   - API 테스트
   - 프론트 컴포넌트 테스트
   - E2E 테스트 필요성
   - 테스트하기 어려운 구조와 개선안
