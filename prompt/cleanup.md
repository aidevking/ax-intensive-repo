### 프로젝트 최종 정리 및 검증 작업

프로젝트 전체를 점검하여 구조를 최적화해 주세요.

#### 1. 불필요한 파일 및 코드 정리

* 현재 프로젝트를 전체 스캔합니다.
* 사용되지 않는 파일, 중복 파일, 임시 파일, 테스트 산출물, 레거시 코드 등을 식별합니다.
* 실제 기능에 영향을 주지 않는 범위에서 제거합니다.
* 쿼드(Quad) 품질 및 기존 기능은 반드시 유지합니다.
* 과도한 리팩토링보다는 유지보수성과 단순화를 우선합니다.

#### 2. 의존성 및 구조 검증

* 사용되지 않는 패키지, 라이브러리, import 문을 정리합니다.
* 디렉터리 구조가 목적에 맞게 구성되어 있는지 검토합니다.
* 불필요한 중첩 구조가 있다면 최소한으로 정리합니다.

#### 3. 테스트 수행

* 전체 테스트를 다시 실행합니다.
* 모든 테스트가 통과하는지 확인합니다.
* 빌드, 린트, 타입 체크(가능한 경우)도 함께 수행합니다.
* 실패 항목이 있다면 원인을 수정한 후 재실행합니다.

#### 4. 결과 보고

다음 내용을 정리하여 보고합니다.

* 삭제된 파일 목록
* 수정된 파일 목록
* 테스트 결과
* 빌드 결과
* 린트 결과
* 타입 체크 결과
* 추가로 발견된 개선 사항

#### 5. README 업데이트

정리 완료 후 README를 최신 상태로 갱신합니다.
다음 내용을 반드시 반영합니다.

* 현재 프로젝트 구조(Tree)
* 주요 디렉터리 및 파일 설명
* 실행 방법
* 개발 환경 설정 방법
* 테스트 실행 방법
* 빌드 방법
* 아키텍처 또는 모듈 구성 설명
* 최근 구조 변경 사항

#### 작업 원칙

* 기능 변경보다 정리(Cleanup)와 안정성 유지에 집중합니다.
* 기존 동작과 API 호환성을 유지합니다.
* 삭제 또는 변경 시 근거를 명확히 기록합니다.
* 코드 품질은 유지하거나 개선하되, 불필요한 복잡성은 제거합니다.
* 최종 상태는 "테스트 통과 + README 최신화"를 기준으로 완료합니다.

**최종 목표:** 기능과 품질은 그대로 유지하면서 프로젝트를 최대한 간결하고 유지보수하기 쉬운 상태로 정리하고, 검증 결과와 최신 구조를 README에 반영합니다.
