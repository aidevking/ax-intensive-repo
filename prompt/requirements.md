## 검토 지침

1. 모든 요구사항을 개별적으로 분석한다.
2. 각 요구사항에 대해 다음 중 하나로 판정한다.

   * 충족 (Satisfied)
   * 부분 충족 (Partially Satisfied)
   * 미충족 (Not Satisfied)
   * 판단 불가 (Insufficient Evidence)
3. 반드시 근거를 제시한다.
4. 추측하지 말고 제공된 정보에 기반하여 판단한다.
5. 누락된 기능, 모호한 구현, 잠재적 리스크를 식별한다.
6. 사용성, 예외 처리, 성능, 보안 측면에서 요구사항이 암묵적으로 위반되는 부분이 있는지 검토한다.

## 추가 분석

가능하다면 다음도 분석하시오.

* 사용자 경험(UX) 관점의 문제점
* 예외 상황 처리 누락
* 비기능 요구사항(성능, 보안, 확장성) 충족 여부
* 요구사항 간 충돌 가능성
* 테스트 케이스 추가 필요 항목

제공된 증거만을 사용하여 객관적으로 평가하시오. 추측성 판단은 "판단 불가"로 표시하시오.
