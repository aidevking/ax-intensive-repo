api-architect 에이전트와 frontend-builder 에이전트를 함께 사용해서
백엔드와 프론트엔드를 실제로 연결하고 검증해줘.

요구사항:
1. 환경설정 통일
   - backend/.env(또는 settings)에 PORT, CORS allow_origins를 명시 변수로 분리
   - frontend/.env(또는 settings)에 BACKEND_URL을 명시 변수로 분리 (하드코딩 금지)
   - 두 .env.example 파일을 만들어서 어떤 값을 채워야 하는지 명확히 한다
2. 동시 실행 스크립트
   - 루트에 run.sh(또는 Makefile, 혹은 docker-compose.yml) 작성:
     backend는 uvicorn으로, frontend는 streamlit run으로 한 번에 기동
   - 포트 충돌이 없는지(예: backend 8000, frontend 8501) 확인
3. CORS 실제 동작 확인
   - 프론트엔드(브라우저)에서 백엔드를 호출했을 때 CORS 에러 없이 응답이 오는지
     실제로 두 서버를 띄운 상태에서 확인 (코드 리뷰가 아니라 실제 호출)

구현 후 다음을 직접 실행해서 검증해줘 (반드시 두 서버를 동시에 띄운 상태에서):
1. run.sh(또는 동등 스크립트)로 백엔드+프론트엔드가 동시에 정상 기동되는지
2. 프론트엔드 사이드바의 "백엔드 연결됨" 표시가 실제로 "연결됨"으로 뜨는지
3. 전체 시나리오를 처음부터 끝까지 실제 클릭으로 실행:
   앱 선택/수집 → (실제로 백엔드가 데이터를 받아 처리) → EDA 화면에 실제 결과 표시
   → 토픽/감성 화면에 실제 결과 표시 → 리포트 생성 버튼 → 실제 백엔드 응답으로 리포트 표시
   - 이 흐름 중 mock이나 샘플 데이터가 아니라 실제 두 서버 간 HTTP 통신으로 이루어지는지 확인
4. 의도적으로 백엔드를 잠깐 내렸다가 다시 올렸을 때 프론트엔드가
   - 죽지 않고 "연결 끊김" 상태를 보여주는지
   - 백엔드가 복구되면 자동 또는 사용자 재시도로 다시 정상 동작하는지
5. 응답 시간이 긴 리포트 생성(/generate/report)이 실제 환경에서도 30초 기준 내에 들어오는지
   (mock이 아닌 실제 LLM/RAG 호출 기준으로 재측정)

검증 결과를 표로 정리해줘. 위 5개 항목이 실제 두 서버 동시 구동 상태에서 전부 통과하기 전까지는
"백엔드-프론트엔드 연동 완료"라고 말하지 마. 미흡한 항목은 어느 쪽(backend/frontend) 어느 파일을
고쳐야 하는지 구체적으로 짚어줘.
