2. 주제 선정 가이드
주제는 자유입니다. 다만 우리는 신한은행 임직원이고, 배운 기술을 실제 업무에 어떻게 쓸
수 있을지 고민하는 것이 이 프로젝트의 진짜 목적입니다.
가능하면 금융·은행 업무와 관련된 주제를 선택해 주세요. 본인 부서의 실제 고민, 고객 응대,
데이터 분석 업무에서 출발하면 가장 좋은 결과물이 나옵니다.
금융 연관 주제 예시 (그대로 써도 되고, 변형·창작 환영):
• 고객 이탈/해지 예측 + 상담원에게 추천 멘트를 제안하는 어시스턴트
• 여신(대출) 부도 예측 + 사내 규정 문서 기반 심사 보조 챗봇
• 카드 이상거래(이상치) 탐지 + 자동 리포트 생성

3. 필수 요구사항 (★ 반드시 포함)
단순히 "Claude Code로 코드를 한 번 생성"하는 수준을 넘어, 배운 기술을 통합하고 그
선택의 이유를 설명할 수 있어야 합니다.
아래 세 가지 기술 축 중 최소 2개 이상을 하나의 시나리오로 엮어 주세요. (3개 모두 권장)
• 데이터 분석 — Pandas 기반 EDA + 전처리 (결측·이상치 처리 판단 포함)
• 머신러닝 — 모델 학습 + 평가지표 제시 + 베이스라인 대비 개선
• RAG / LangChain / LLM — 분석 결과를 자연어로 설명하거나 문서 기반 Q&A 제공
추가 필수 조건:
• 데모 앱: Streamlit 또는 Vercel 등으로 실행(시연 가능한) 화면 제출
• 검증: "잘 돌아간다"가 아니라 "결과가 맞다"를 근거로 증명 (오답/실패 케이스 분석 포함)
• 역할 분담 명시: 4명이 각자 무엇을 했는지 기록

금융데이터거래소 등)을 다운로드하거나, 배운 웹 스크래핑 기술로 직접 수집하세요.
스크래핑 시 사이트 이용약관과 robots.txt를 반드시 확인하고 과도한 요청은 삼가세요.

가상 데이터 생성도 가능합니다.
